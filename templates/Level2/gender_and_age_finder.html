Definitions

HTTP Requests

HTTP requests are messages sent by a client (like your browser or a mobile app) to a server. They ask the server to perform specific actions, like fetching data, submitting a form, or deleting a resource. There are several types of HTTP requests, but the most common ones are:

  • GET: Retrieve information from the server.

  • POST: Send data to the server (e.g., form submission).

  • PUT / PATCH:Update existing information on the server.

  • DELETE: Remove information from the server.


API

An API is a set of rules that allows different software applications to communicate with each other. Think of it as a messenger that takes requests, tells a system what to do, and returns the response. APIs can be used for anything from integrating third-party services (like payment gateways) to pulling data from a database.

In essence:

  • HTTP requests are the way we send and receive data over the web.

  • APIs are the interfaces that allow different software systems to communicate using those HTTP requests.

Imagine wanting to check the weather from an app: the app sends an HTTP request to a weather API, which then returns the weather data.


API sources

Genderize API

Genderize API

A simple API that checks the gender of a person based on a first name The Genderize API allows you to check the statistical probability of a name being male or female

The API exposes a single endpoint that accepts a name and returns a payload with the prediction. It uses query parameters for requests and returns JSON-encoded responses.

Agify API

A simple API that estimates the age of a person based on a first name The Agify API allows you to estimate the age of a person based on their first name.

The API exposes a single endpoint that accepts a name and returns a payload with the prediction. It uses query parameters for requests and returns JSON-encoded responses.
